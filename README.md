## LegadoDigital - Acceptance Tests

Este repositorio contiene los archivos ".feature" elaborados para los Acceptance Tests del producto LegadoDigital.

Los escenarios fueron redactados en lenguaje Gherkin, utilizando la estructura Dado que / Cuando / Entonces, con el objetivo de validar las User Stories seleccionadas del proyecto.

## Estructura del repositorio

- `features/ep02_gestion_activos_documentos.feature`
- `features/ep04_permisos_reglas_acceso.feature`
- `features/ep09_seguridad_trazabilidad.feature`

## Épicas consideradas

- EP02: Gestión de activos y documentos digitales.
- EP04: Configuración de permisos y reglas de acceso.
- EP09: Seguridad y trazabilidad.

## Historias funcionales evaluadas

- US06: Registrar activo digital.
- US10: Cargar documento importante.
- US20: Asignar nivel de acceso.
- US21: Seleccionar información compartida.

## Historias no funcionales evaluadas

- US45: Activar autenticación en dos pasos.
- US46: Validar inicio en dispositivo nuevo.
- US47: Consultar historial de accesos.
- US48: Proteger información mediante cifrado.
